## Immediate implementation goal

Deliver the first exact-gated live texture compatibility pilot with complete fallback telemetry and packaged proof.

The pilot should reuse a verified prepared texture artifact at the narrowest reviewed image seam while preserving the remainder of Starsector's original texture path.

### Required behavior

1. The target activates only when all manually reviewed identity fields match:
   - class SHA-256;
   - required method descriptors;
   - source archive SHA-256 and suffix;
   - source kind;
   - loader class and loader name.
2. Unknown or changed installations remain untouched.
3. A valid cache hit returns semantically equivalent data.
4. Any miss, stale record, checksum failure, unsupported image, ambiguity, internal exception, or circuit-breaker state calls the original method.
5. The game installation, launcher, mods, saves, and VM parameter files are never edited.
6. The existing environment/property kill switch remains authoritative.
7. The adapter emits bounded hit, miss, fallback, corruption, and disable-reason counters.

## Suggested first PR decomposition

### PR A: benchmark scenario and telemetry contract

Before claiming a live gain, add a machine-readable benchmark result schema and a scenario protocol that separates:

- process to main menu;
- main menu to campaign readiness;
- campaign to first combat readiness.

Keep execution manual initially if reliable UI automation is unavailable. The CLI should still collect run identity, mode, milestone timestamps supplied by the runner, cache state, adapter counters, and exit status into one comparable result.

Acceptance:

- deterministic JSON schema;
- fixture validation;
- explicit cold/warm/build/corrupt/changed-profile modes;
- no performance claim.

### PR B: texture compatibility pilot

Implement one exact transformation plan using the existing adapter registry and packaged agent.

Acceptance:

- exact target fixture accepted;
- one-field mismatch fixtures rejected;
- cold miss calls original once;
- valid warm hit uses prepared data;
- corrupt hit calls original once and repairs or quarantines safely;
- unknown build applies zero transformations;
- process-local kill switch disables the target;
- packaged child-JVM test uses the shaded agent JAR;
- full Linux/macOS/Windows gates pass.

### PR C: upload-ready texture path

After the compatibility pilot works, bind the lower `BufferedImage -> ByteBuffer` seam to the existing prepared-pixel blob so a hit bypasses decode/conversion work while preserving original OpenGL upload and texture lifetime.

Acceptance additionally requires pixel, dimensions, alpha, orientation, and derived-color equivalence plus bounded direct-buffer ownership tests.

## Benchmark protocol for the first live target

Use a stable updated-mod profile that can complete the scenario and exit normally.

For each mode, collect at least five successful runs:

1. OFF cold-ish;
2. OFF warm;
3. ENABLED build/miss;
4. ENABLED warm hit;
5. ENABLED corrupt artifact;
6. ENABLED after changed source or mod order.

Report every run plus median/min/max. Do not delete failed runs; classify their cause.

Minimum metrics:

- main-menu and campaign-readiness time;
- texture hits, misses, fallback reasons, and bytes served;
- original image-decode and conversion work remaining in JFR;
- peak heap and direct-buffer use;
- cache bytes and entry count;
- normal exit status.

### Audio

1. Build installed-JOrbis equivalence fixtures.

3. Preserve stream ownership and exception behavior.
4. Implement short fully decoded effect reuse first.
5. Keep streaming music ineligible until proven safe.

### Janino

1. Instrument source/resource lookup during one `generateBytecodes()` operation.
2. Capture the complete ordered dependency context.
3. Prove inner/anonymous class-map completeness.
4. Bind the existing SPJB wrapper only after exact-key completeness tests.
5. A partial or uncertain dependency set is always a miss.

### Shared artifact runtime

Avoid three independent storage layers. Consolidate common manifest, checksumming, quarantine, bounds, cleanup, and telemetry behavior when the first two consumers reveal the stable shared API.

## Prohibited shortcuts

Do not:

- generate allowlists automatically from probe output;
- weaken exact identity gates to class names or method names alone;
- patch the Starsector installation or launcher;
- swallow original exceptions on a fallback path;
- return partial Janino class maps;
- key Janino only by source size or modification time;
- cache streaming music as fully decoded PCM without policy proof;
- perform OpenGL work on background threads;
- add unbounded maps, logs, artifact counts, buffers, or worker pools;
- claim acceleration from sampled CPU percentages;
- use the two crash-ending traces as final wall-time benchmarks.
